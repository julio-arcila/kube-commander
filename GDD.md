# Kube Commander — Game Design Document

## 🎯 Concept

A **card-based strategy game** that teaches Kubernetes concepts through gameplay. Players deploy cards representing Kubernetes resources (Pods, Services, Deployments, etc.) to handle incoming traffic waves while managing limited CPU and memory.

## 🎮 Gameplay Loop

```
1. Draw cards → 2. Deploy to cluster → 3. Handle traffic waves
       ↓                                      ↓
4. Learn k8s concepts ← 5. Complete level → 6. Earn coins + rewards
```

## 📊 Core Mechanics

### Resource Management
- **CPU Cores** — Each card costs CPU to deploy
- **Memory (RAM)** — Each card costs memory to deploy
- **Score** — Earned by successfully handling traffic

### Card System
Cards map 1:1 to Kubernetes concepts:
- **Compute** (Pod, Deployment, ReplicaSet, StatefulSet, DaemonSet)
- **Network** (Service, Ingress)
- **Storage** (PersistentVolumeClaim)
- **Config** (ConfigMap, Secret)
- **Security** (NetworkPolicy, RBAC) — unlockable

### Progression
- 10 levels, each teaching 1-2 Kubernetes concepts
- Level 1: Pod basics → Level 10: Full production cluster
- Cards unlock as concepts are learned
- Star rating per level (1-3 based on efficiency)

## 🎨 Art Style

**Cyberpunk Datacenter** — Neon-lit server racks, holographic cards, terminal-green UI elements. Think "Blade Runner meets Kubernetes dashboard."

## 📱 Monetization

| Ad Type | Trigger | Frequency |
|---------|---------|-----------|
| Rewarded Video | "Get 50 extra CPU cores" | Player opt-in |
| Interstitial | Between levels (every 3 levels) | Max 1 per 5 min |
| Banner | Bottom of menu screen | Persistent (light) |

## 🏗️ Technical Architecture

```
project.godot
├── scripts/
│   ├── game/
│   │   ├── game_manager.gd    # Autoload — game state, saves
│   │   ├── card_database.gd   # Autoload — all k8s cards
│   │   └── game_board.gd      # Main gameplay scene
│   ├── ui/
│   │   ├── main_menu.gd       # Title screen + level select
│   │   ├── kube_pedia.gd      # Encyclopedia of learned concepts
│   │   └── card.gd            # Card UI component
│   └── systems/
│       ├── ad_manager.gd      # AdMob integration
│       └── save_system.gd     # JSON save/load
├── scenes/
│   ├── game/card.tscn, game_board.tscn
│   ├── ui/main_menu.tscn, kube_pedia.tscn
│   └── levels/ (generated)
└── assets/
    ├── sprites/ (AI-generated or hand-drawn)
    ├── audio/ (AI-generated or CC0)
    └── fonts/ (JetBrains Mono for terminal feel)
```

## 🧠 AI Integration Opportunities

- **YAML snippets** — Real k8s manifests on each card

- **Sprite generation** — DALL-E / SD for card art
- **Music/SFX** — AI-generated audio
- **Kube-pedia** — LLM-powered Q&A about k8s concepts you've learned
